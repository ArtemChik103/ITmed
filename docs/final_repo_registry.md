# Финальный реестр состава репозитория

Этот реестр фиксирует, что именно должно остаться в `main` для финальной экспертной проверки.

## Оставить

- `api/`
- `core/`
- `plugins/`
- `frontend/`
- `scripts/`, которые нужны для demo, submission и финальной выгрузки
- `tests/`
- `README.md`
- `Dockerfile`
- `docker-compose.yml`
- `start.sh`
- `requirements.txt`
- `deliverables/`
- минимальные `docs/`, которые помогают запустить проект и понять ограничения решения

## Не добавлять в финальный `main`

- временные research-заметки и промежуточные phase-документы;
- локальные analysis outputs, на которые больше не опирается runtime;
- служебные build-папки и временные bundle-архивы;
- тяжелые локальные датасеты и промежуточные артефакты обучения;
- все материалы, которые не нужны эксперту для запуска, проверки и защиты.

## Практическое правило

- если файл нужен для запуска backend/frontend, локального runtime, проверки `test_done`, генерации PDF или объяснения ограничения по geometry, он остается;
- если файл нужен только как промежуточная исследовательская история, он не должен попадать в финальный публичный состав;
- `predictions.csv`, `results_test_done.zip` и `presentation.pdf` считаются частью финального deliverables-пакета.
