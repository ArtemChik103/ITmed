# Задача по реализации проекта 
1. Общее описание 
Цель — разработать интеллектуальную систему для диагностики патологий по медицинским изображениям (рентгенографическим снимкам).
Система должна решать две задачи: 
- Образовательную. Визуализация процесса анализа для обучающихся. Демонстрация того, на основе чего сделан вывод. 
- Клиническую. Помощь врачу в диагностике через классификацию (есть патология, нет патологии) и количественные расчеты параметров, необходимых для постановки диагноза.  

Исходные данные.
Пароли и архивы опубликованы в телеграмм-канале Хакатона.
Используйте предоставленные данные только в целях предусмотренных хакатоном. Распространение допускается только среди зарегистрированных команд хакатона.

2. Требования 
2.1 Расширяемость 
Главная техническая ценность проекта – расширяемость. Система должна быть спроектирована как платформа, в которую можно легко добавлять новые модули анализа для других анатомических зон или патологий без переписывания ядра. 
2.2. Модуль машинного обучения (МО) 
Модуль должен работать как 
Классификатор. Определение наличия/отсутствия патологии. 
Метрический анализатор. Автоматическое нахождение ключевых точек и расчет углов/расстояний и других необходимых количественных показателей. 
2.3. Интерфейс пользователя 
Режим врача. Врач загружает изображение в программу. Программа анализирует изображение, выводит результат (ответ модуля машинного обучения): расчетные значения, уверенность в определении патологии или другие важные величины. Программа показывает дополнительные построения на изображениях если необходимо. 
Режим обучения (образовательный режим). Подробная визуализация. Отображение опорных точек, линий и пояснений (при необходимости), объяснение почему система пришла к такому выводу. 
4. Критерии оценки проектов 
4.1 Общие критерии
Проект оценивается по нескольким аспектам, для каждого задан относительный вес.
1. Архитектура и структура решения — 25%
Насколько легко добавить новый тип изображений? Оценивается чистота кода, модульность, расширяемость. 
2. Точность анализа — 25% 
Метрики модели на тестовом датасете. 
3. Удобство использования — 20% 
Насколько интерфейс понятен, удобен и на сколько быстро в нём можно работать Наглядность расчетных параметров и зон интереса (если есть). 
4. Презентация и защита — 15% 
o Ясность и структурированность презентации. 
o Описание архитектуры, функционала. 
o Сценарии работы 
o Ответы на вопросы жюри. Демонстрация рабочего прототипа. 
5. Бонусные баллы — 15% 
Оригинальность идеи, реализации
